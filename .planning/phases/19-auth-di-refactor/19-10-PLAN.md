<objective>
T-19-10 — migrate the 14 integration test fixtures from `Container().db_session_factory.override(...)` + `dependencies.set_container(container)` + `app.add_middleware(DualAuthMiddleware, container=container)` to the new shape `app.dependency_overrides[get_db] = override_get_db`.

This plan MUST land BEFORE Plans 11-13 delete the DualAuthMiddleware / CsrfMiddleware / container modules. The atomic-commit invariant from CONTEXT.md ("Tests must be green at each step. ... so any commit is independently revertable") requires that pytest collection succeeds at every commit. Deleting `app/core/dual_auth.py` while integration tests still import `DualAuthMiddleware` for fixture middleware-stack assembly would break collection — pytest would error before any tests ran.

By switching the fixture SURFACE to dependency_overrides BEFORE the deletions, downstream waves can delete the source modules without breaking collection. Symbolic imports of `Container` / `DualAuthMiddleware` may still be needed in some fixtures at this wave (modules still exist) — that is acceptable; the critical change is that the fixture no longer DEPENDS on those modules being mounted as middleware, only on the production app being driven via dependency_overrides.

Purpose: REFACTOR-06 — preserve test inventory through the structural refactor; the atomic-commit invariant holds across Plans 11-13.

Output: 14 integration test files migrated + tests/fixtures/test_container.py edited or stubbed for deletion in Plan 13.
</objective>
