    **useUploadOrchestration.ts** -- Add cancel support, retrying state, and classified error propagation:

    1. **Abort ref map:** Add `const abortControllerRef = useRef<Map<string, () => void>>(new Map())` to track abort functions per file ID. This replaces the need for a single abort ref since we process files sequentially but need to cancel by file ID.

    2. **Retrying state:** Add state to track which file is currently retrying:
       - `const [retryingFileId, setRetryingFileId] = useState<string | null>(null)`.
       - This will be passed to the UI to show the "Retrying..." indicator.

    3. **Update processViaTus:** After calling `startTusUpload`, store the abort function:
       - `startTusUpload` returns `{ abort }` synchronously (09-01 keeps it synchronous -- the async resume+start runs internally via fire-and-forget). No await needed.
       - Store the abort function immediately:
         ```
         const { abort } = startTusUpload(item.file, metadata, {
           onProgress: ...,
           onSuccess: ...,
           onError: ...,
           onRetrying: () => setRetryingFileId(item.id),
         });
         abortControllerRef.current.set(item.id, abort);
         ```

    4. **Update onError callback in processViaTus:**
       - 09-01 already migrated the onError signature to `(userMessage, technicalDetail, isRetryable)` and updated the call site to use `setFileError(item.id, userMessage, technicalDetail)`.
       - Add cleanup: clear retrying state `setRetryingFileId(null)` and abort ref `abortControllerRef.current.delete(item.id)`.

    5. **Update onSuccess callback in processViaTus:**
       - Clear retrying state: `setRetryingFileId(null)`.
       - Clear abort ref: `abortControllerRef.current.delete(item.id)`.

    6. **Update onProgress callback in processViaTus:**
       - Clear retrying state on each progress update: call `setRetryingFileId(null)` in onProgress. React will no-op if already null, avoiding stale closure issues.

    7. **handleCancel callback:**
       ```
       const handleCancel = useCallback((id: string) => {
         const abort = abortControllerRef.current.get(id);
         if (abort) {
           abort();  // calls upload.abort(true) -> DELETE + localStorage cleanup
           abortControllerRef.current.delete(id);
         }
         setRetryingFileId(null);
         updateFileStatus(id, 'pending');
         updateFileProgress(id, 0, 'uploading');
         updateFileUploadMetrics(id, '', '');
         // Clear processing refs if this was the active file
         if (currentFileIdRef.current === id) {
           currentFileIdRef.current = null;
           currentTaskIdRef.current = null;
         }
       }, [updateFileStatus, updateFileProgress, updateFileUploadMetrics]);
       ```

    8. **Expose in return value:**
       - Add `handleCancel` and `retryingFileId` to the return object.
       - Update `UseUploadOrchestrationReturn` interface to include:
         - `handleCancel: (id: string) => void`
         - `retryingFileId: string | null`
  </action>
