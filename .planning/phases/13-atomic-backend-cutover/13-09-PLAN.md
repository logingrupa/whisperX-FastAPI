ASGI middleware ordering reverses registration order: LAST registered runs FIRST on request. Required order (request flow):
1. CORS (pre-flight handling) — registered LAST
2. DualAuthMiddleware (auth resolution) — registered SECOND
3. CsrfMiddleware (post-auth CSRF check) — registered FIRST

So registration order is: CsrfMiddleware → DualAuthMiddleware → CORSMiddleware.
